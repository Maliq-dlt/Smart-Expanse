## 2024-05-01 - Icon-Only Button Accessibility Pattern
**Learning:** Found a pattern where icon-only buttons (like those in TopBar.tsx) lack comprehensive accessibility. While `aria-label` is sometimes present, native hover tooltips (`title`), keyboard focus visibility (`focus-visible` ring), and `aria-hidden="true"` on the inner icon elements are missing. This makes them hard to discover via mouse hover, confusing for screen readers interpreting ligature icons (e.g. material-symbols-outlined reading out "notifications"), and untrackable for keyboard navigators.
**Action:** Standardize icon-only buttons to always include: 1) `aria-label` for screen readers, 2) `title` for native mouse tooltips, 3) `focus-visible:ring-2 focus-visible:ring-[var(--color-primary)] outline-none` for keyboard focus, and 4) `aria-hidden="true"` on the inner icon element itself.
